1 Translation with Push-Down Automata {#pda}
2 =====
3 
4 This HiFST package can also perform decoding using Push-Down Automata (PDA) as described
5 in [\ref Iglesias2011, \ref Allauzen2014]. We call this the HiPDT decoder. A brief overview of it and an example on how to use it is given next.
6 
7 In this framework, the RTN at the top-most cell of the CYK grid is
8  converted into a PDA (via the Replace operation in the \ref OpenFst Extensions)
9 and efficiently
10  composed with a language model to produce another PDA. To obtain the
11  final output FSA, the PDA is expanded to an FSA either entirely (if
12  memory is sufficient and the language model is small enough) or via pruned expansion (for larger language models). This is useful in
13  exploring large and complex translation grammars where HiFST requires
14  a lot of local pruning. However, it requires the language model to be
15  'small' (for example, entropy-pruned as in
16  [\ref Iglesias2011, \ref Allauzen2014]). Therefore, we then typically rescore the pruned
17  output FSA with a stronger language model.
18 
19 This whole setup can be accomplished in one single command as follows:
20 
21  > hifst.O2 --config=configs/CF.hiero.pdt &> log/log.hiero.pdt
22 
23 Please see the config file for the parameters needed, along with explanatory comments.
24 
25 In this particular example, the output 1-best for the first two
26 sentences is identical to the baseline Hiero case (with translation
27 via RTN replacement followed by composition). However, the output lattices differ as they contain
28 different hypotheses (due to the different pruning strategy).

30 \section pda_rtns Recursive Transition Networks
31 
32 As already discussed with respect to \ref lpruning,
33 HiFST generates an initial representation of the space of translation in the form of an RTN,
34 which is then transformed either to WFSAs or to PDAs using the
35 \ref OpenFst [Replace](http://openfst.org/twiki/bin/view/FST/ReplaceDoc) operation
36 prior to application of the language model.
37 
38 HiFST can save the RTNs to disk, and the language model application and shortest path operations can be carried out using the \ref OpenFst command line tools.
39 
40 For example, consider generation of the baseline lattices with the Shallow-1 translation grammar and the 4-gram language model. The command can be re-run, but with added instructions to save the RTNs to disk:
41 
42  > hifst.O2 --config=configs/CF.baseline --hifst.writertn=output/exp.baseline/rtn/?/%%rtn_label%%.fst --grammar.storentorder=output/exp.baseline/rtn/ntmap --hifst.rtnopt=yes &> log/log.baseline.rtn
43 
44 The RTNs are written to the directory `output/exp.baseline/rtn/*` as
45 
46  > ls output/exp.baseline/rtn/1/
47  1001000007.fst 1003001002.fst 1003003000.fst 1003004001.fst 1003006000.fst 1004002000.fst 1004005000.fst
48  1003001000.fst 1003002000.fst 1003003001.fst 1003005000.fst 1003007000.fst 1004003000.fst 1004006000.fst
49  1003001001.fst 1003002001.fst 1003004000.fst 1003005001.fst 1004001000.fst 1004004000.fst
50 
51 and the non-terminal mapping is written to output/exp.baseline/rtn/ntmap :
52 
53  > cat output/exp.baseline/rtn/ntmap
54  S 1
55  D 2
56  X 3
57  V 4
58 
59 Each RTN name is of the form 1ABC.fst , where A, B, and C are 3-digit strings.
60  - A is the numerical code for a non-terminal in the grammar; in this case, 001 corresponds to S.
61  - B indicates a position in the source sentence: 0 <= B < I, where I is the source sentence length
62  - C indicates the offset to the end of a span, 0 <= C and B+C < I
63  - The automata 1ABC.fst corresponds to T_(A,B,B+C) in the formulation of \ref lpruning
64 
65 In this example, the automata 1003001002.fst contains all derivations headed by X, the third non-terminal, and spanning source positions 1 to 3 (=1+2).
66 
67 The root automata is 1001000007.fst, since A=001 (the S non-terminal),
68 B=0 (the first source position), and C=7 (the sentence has 8 words).
69 This automata is a representation of all possible translations of the
70 source sentence under this grammar, as can be seen by printing its paths (here the first 2):
71 
72  > cat output/exp.baseline/rtn/1/1001000007.fst | printstrings.O2 --semiring=lexstdarc -u --nbest=2 2>/dev/null
73  1 1004001000 11 384 1004003000 1004004000 1004005000 1004006000 2
74  1 1004001000 11 384 1003003001 1004005000 1004006000 2
75 
76 
77 As can be seen, the symbols are a mix of target language symbols (1,2,11,384,...) and pointers to other automata (1004001000, 1004003000, ...).
78 Conversion of the RTN is done by recursive substitution of these symbols by the FSTs to which they point, starting from the root automata.

80 \section pda_replace Replacement: Translation by Converting RTNs to WFSAs
81 
82  > fstreplace | head -n 3
83  Recursively replaces FST arcs with other FST(s).
84 
85  Usage: fstreplace root.fst rootlabel [rule1.fst label1 ...] [out.fst]
86 
87 Note that the root FST and root label are always of the form 1001000C, where C = I - 1 , where I is the source sentence length.
88 
89 HiFST uses the same names for
90 rules and labels, so we get get the filenames and labels in the right form by, e.g.
91 
92  > ls output/exp.baseline/rtn/1/1*.fst | sed 's,\(.*\)/\(.*\).fst,\1/\2.fst \2,' | head -n 5
93  output/exp.baseline/rtn/1/1001000007.fst 1001000007
94  output/exp.baseline/rtn/1/1003001000.fst 1003001000
95  output/exp.baseline/rtn/1/1003001001.fst 1003001001
96  output/exp.baseline/rtn/1/1003001002.fst 1003001002
97  output/exp.baseline/rtn/1/1003002000.fst 1003002000
98 
99 The following will expand the RTN into an FSA:
100 
101  > fstreplace `ls output/exp.baseline/rtn/1/1*.fst | sed 's,\(.*\)/\(.*\).fst,\1/\2.fst \2,'` > output/exp.baseline/rtn/1/T.fst
102 
103 The WFSA T is the replacement of the RTN that was generated in translation.

105 \section rtn_lm_app Composition and Shortest Path
106 
107 The applylm tool can be used to apply the baseline 4-gram language model to T via composition. This generates a new WFSA containing both translation and language model scores:
108 
109 - Input
110  - M/lm.4g.mmap : n-gram LM
111  - output/exp.baseline/rtn/1/T.fst : WFSA containing translation scores
112 - Output
113  - output/exp.baseline/rtn/1/TG.fst.gz : WFSA containing translation grammar and LM scores
114 
115 The output is written in the form of a transducer, with the RTN labels as the input symbols and the target language words on the output symbols:
116

150 \section pda_expand Expansion: Translation by Composition of PDAs and WFSAs followed by Pruned Expansion
151 
152 
153 When using PDTs, the decoding process differs from the above in that the RTN is not expanded to an FSA prior to composition with the LM. Instead, HiPDT replaces the RTN by a PDA, which is efficiently composed with the LM to produce another PDA. Finally, this resulting PDA is converted to the final translation FSA via pruned expansion.
154 
155 As explained in the example of \ref pda, this process is done by the decoder in one go. For explanatory reasons, here we reproduce it externally for one sentence via the command line.
156 
157 (Note: as of March 2014 we have not managed to make the OpenFST PDT command line operations work well with our lexicographic library, so the standard tropical semirring must be used in this example. This makes it slightly more complex than it should be, but we believe it is useful to understand HiPDT anyway)
158 
159 First, we dump the RTN for the full hiero grammar as follows:
160 
161  > hifst.O2 --config=configs/CF.hiero --hifst.writertn=output/exp.hiero/rtn/?/%%rtn_label%%.fst --grammar.storentorder=output/exp.hiero/rtn/ntmap --hifst.rtnopt=yes &> log/log.hiero.rtn
162 
163 Then, we ensure that the RTN files are in the tropical semiring:
164 
165  > mkdir -p output/exp.hiero/rtn-tp/1 output/exp.hiero/rtn-tp/2
166  > pushd output/exp.hiero/rtn/ ; for f in ?/100*.fst; do cat $f | lexmap.O2 --action=lex2std > ../rtn-tp/$f; done; popd
167 
168 The PDT is then created as follows:
169 
170  > for f in 1 2; do pdtreplace --pdt_parentheses=output/exp.hiero/rtn-tp/$f/parens.txt `ls output/exp.hiero/rtn-tp/$f/1*.fst | sed 's,\(.*\)/\(.*\).fst,\1/\2.fst \2,'` > output/exp.hiero/rtn-tp/$f/T.pdt; done
171 
172 where the `pdt_parentheses' option indicates that the open/close parentheses symbols are to be stored into a file. This is used later in order to expand the PDA to an FSA.
173 
174 Then the PDA is composed with the weak language model. This is done via the standard composition algorithm, but making sure that open/close parentheses symbols are treated as epsilons by the composition algorithm. To accomplish this, their respective output symbols need to be relabel to 0 before applying the LM:
175 
176  > for f in 1 2; do cat output/exp.hiero/rtn-tp/1/parens.txt | tr '\t' '\n' | sed 's/$/\t0/' > output/exp.hiero/rtn-tp/$f/parens-to-epsilon.txt ;
177  > cat output/exp.hiero/rtn-tp/$f/T.pdt | fstrelabel -relabel_opairs=output/exp.hiero/rtn-tp/$f/parens-to-epsilon.txt > output/exp.hiero/rtn-tp/$f/Tb.pdt
178  > applylm.O2 --lm.load=M/lm.4g.eprnd.mmap --lattice.load=output/exp.hiero/rtn-tp/$f/Tb.pdt --lattice.store=output/exp.hiero/rtn-tp/$f/TG.pdt ;
179  > done
180 
181 Then the resulting PDT is expanded into an FSA while applying a pruning weight of 9:
182 
183  > for f in 1 2; do fstproject output/exp.hiero/rtn-tp/$f/TG.pdt | pdtexpand --pdt_parentheses=output/exp.hiero/rtn-tp/$f/parens.txt --weight=9 > output/exp.hiero/rtn-tp/$f/TG.fst ; done
184 
185 Finally, the weak LM is removed and the full LM is applied:
186 
187  > for f in 1 2; do applylm.O2 --lm.load=M/lm.4g.eprnd.mmap --lm.featureweights=-1 --lattice.load=output/exp.hiero/rtn-tp/$f/TG.fst --lattice.store=output/exp.hiero/rtn-tp/$f/TG-nolm.fst ; applylm.O2 --lm.load=M/lm.4g.mmap --lattice.load=output/exp.hiero/rtn-tp/$f/TG-nolm.fst --lattice.store=output/exp.hiero/rtn-tp/$f/TG-final.fst ; done
188 
189 The final FSA that results from this process (`output/exp.hiero/rtn-tp/1/TG-final.fst`) should be equivalent to the one obtained by HiPDT (`output/exp.hiero.pdt/LATS/1.fst.gz`) except for numerical differences. Their 1-best hypothesis can be obtained as follows:
190 
191  > zcat output/exp.hiero.pdt/LATS/1.fst.gz | printstrings.O2 --semiring=lexstdarc -w -m wmaps/wmt13.en.wmap 2>/dev/null
192  <s> the republican strategy of resistance to the renewal of obama 's election </s> 55.2515,-11.6445
193 
194  > cat output/exp.hiero/rtn-tp/1/TG-final.fst | printstrings.O2 -w -m wmaps/wmt13.en.wmap 2>/dev/null
195  <s> the republican strategy of resistance to the renewal of obama 's election </s> 55.2515
