1 Installation {#build}
2 =====================
3 
4 This section contains instructions on downloading, compiling, and
5 installing the UCAM-SMT tools.
6 
7 There are three sets of tools that can be installed:
8 
9  * \ref install_binaries -- HiFST C++ binaries and dependencies (OpenFst, KenLM, ...)
10  * \ref tutorial_install -- HiFST tutorial based on the UCAM WMT'13 Russian-English system [\ref Pino2013]
11  * \ref rulextract_install -- Hadoop-based translation grammar extraction
12 
13 The \ref install_binaries is **required** to run the \ref
14 tutorial_. However the \ref tutorial_ does **not require** the
15 translation grammar - translation grammars, language models, etc., are
16 provided in the tutorial. However, the Hadoop-based translation
17 grammar extraction may be useful if you wish to extract your own
18 translation grammars from aligned parallel text.

20 \section install_binaries HiFST Binary Installation
21 
22 **Note**:The following instructions are for the Bash shell.
23 
24 The code can be cloned from the following GitHub address:
25 
26  > git clone https://github.com/ucam-smt/ucam-smt.git
27 
28 In the following, `HiFSTROOT` designates the cloned directory,
29 i.e. the following should be a complete path to the cloned directory
30 
31  > export HiFSTROOT=complete_path_to_hifst_cloned_directory
32 
33 As a check, the following command should find the `README.md` file downloaded from github:
34 
35  > ls $HiFSTROOT/README.md
36 
37 Once downloaded, the tools and libraries are compiled in the directory
38 `$HiFSTROOT` by running the script `build-test.sh`.
39 
40 **Note** that the
41 environment variable `TGTBINMK` specific compilation option
42 (e.g. optimisation level, static vs dynamic, etc.) can be set to a value of your choosing
43 before running the script. See `build-test.sh` for the supported options.
44 
45  > cd $HiFSTROOT
46  > export TGTBINMK=O2 # change this as appropriate for your environment. If you omit this line, the default is `export TGTBINMK=O2`.
47  > ./build-test.sh
48 
49 This should download and install necessary dependencies,
50 compile the code and run tests. The `README.md` in the cloned directory also
51 contains useful information for the installation.

53 \section hifst_paths HiFST Paths and Environment Variables
54 
55 After HiFST is successfully built and tested, the file $HiFSTROOT/Makefile.inc
56 will contain environment variable settings needed to run the HiFST
57 binaries and the OpenFST tools using the HiFST libraries. To set these,
58 simply run
59 
60  > source $HiFSTROOT/Makefile.inc
61  > export PATH=$HiFSTROOT/bin:$OPENFST_BIN:$PATH
62  > export LD_LIBRARY_PATH=$HiFSTROOT/bin:$OPENFST_LIB:$BOOST_LIB:$LD_LIBRARY_PATH
63 
64 You should make sure that $HiFSTBINDIR is added first on the path and
65 the library path and that it preceeds the OpenFst directories.
66 If the LD\_LIBRARY\_PATH variable is not set correctly, you will see the message
67 
68  ERROR: GenericRegister::GetEntry : tropical_LT_tropical-arc.so: cannot open shared object file: No such file or directory
69  ERROR: ReadFst : unknown arc type "tropical_LT_tropical" : standard input
70 
71 Sourcing `Makefile.inc` sets the environment variable
72 `TGTBINMK` to point to the HiFST binaries; for example, the following should find the main HiFST binary:
73 
74  > ls $HiFSTROOT/bin/hifst.${TGTBINMK}.bin
75 
76 It is possible to use multiple builds of HiFST by changing the `TGTBINMK` variable.

78 \section rulextract_install Installation of the Hadoop-based Grammar Extraction Tools
79 **Note:** These are not needed to run the basic HiFST \ref tutorial_.
80 
81 Requirements for building/running the rule extraction code:
82  + an Internet connection
83  + preferably 64-bit linux
84  + [sbt](http://www.scala-sbt.org/)
85  + java 1.7+
86 
87 If you don't know how to install sbt, here's one way to do it
88 on Ubuntu:
89 
90  > wget http://dl.bintray.com/sbt/debian/sbt-0.13.5.deb
91  > sudo dpkg -i sbt-0.13.5.deb
92 
93 If you're not root or sudo, you can follow these
94 [instructions](http://www.scala-sbt.org/0.13/tutorial/Manual-Installation.html)
95 
96 If you don't know how to install java, here's one way to install
97 java 7 on Ubuntu, as described
98 [here](http://www.webupd8.org/2012/01/install-oracle-java-jdk-7-in-ubuntu-via.html) (for java 8, see [here](http://www.webupd8.org/2012/09/install-oracle-java-8-in-ubuntu-via-ppa.html)):
99 
100  > sudo add-apt-repository ppa:webupd8team/java
101  > sudo apt-get update
102  > sudo apt-get install oracle-java7-installer
103 
104 If you're not root or sudo, simply download
105 JDK and update the `PATH` and `JAVA_HOME`
106 accordingly.
107 
108 **Note**: the java version used to run your Hadoop cluster should be
109 greater or equal to the java version used to compile the code; otherwise
110 you may get an "Unsupported major.minor version" error.
111 
112 The rule extraction code will be in the `$HiFSTROOT/java/ruleXtract`
113 directory. The variable `$RULEXTRACT` designates this directory
114 from now on:
115 
116  RULEXTRACT=$HiFSTROOT/java/ruleXtract
117 
118 To build the rule extraction software, simply run the
119 following commands:
120 
121  > cd $RULEXTRACT
122  > sbt package
123 
124 You will obtain a jar file named `ruleXtract.jar`
125 located at `$RULEXTRACT/target/ruleXtract.jar` .
126 The variable `$RULEXTRACTJAR` designates this
127 jar from now on:
128 
129  > RULEXTRACTJAR=$RULEXTRACT/target/ruleXtract.jar
130 
131 To run unit tests, simply run:
132 
133  > cd $RULEXTRACT
134  > sbt test
135 
136 If all goes well, you should see a similar looking output:
137 
138  [info] Passed: Total 1, Failed 0, Errors 0, Passed 1
139  [success] Total time: 3 s, completed 12-Aug-2014 10:49:51

142 \section tutorial_install Tutorial Installation
143 
144 Files for this tutorial can be downloaded from the following GitHub address:
145 
146  > git clone https://github.com/ucam-smt/demo-files.git
147  > cd demo-files; gunzip wmaps/*.gz ## Uncompress big wordmap files.
148 
149 **Supplementary Files** There are additional files which can be
150 downloaded from
151 <http://mi.eng.cam.ac.uk/~wjb31/data/hifst.release.May14/> . These
152 are large grammars and language models to be used to replicate
153 evaluation results. They're needed for some of the tutorial steps,
154 e.g. MERT, but aren't included in the default tutorial downloads, due
155 to their size.
156 
157 The tutorial files and sources for this tutorial will be
158 in the `demo-files` directory. The variable `$DEMO`
159 designates this directory from now on. As an optional
160 step, if you wish to regenerate the HTML code for this
161 tutorial, make sure you have doxygen (1.8+
162 for markdown support) and
163 latex (for formulas) installed and run the following commands:
164 
165  > cd $DEMO/Docs.dox
166  > doxygen
167 
168 For this tutorial, it is assumed that rule extraction commands
169 are run from the `$DEMO` directory:
170 
171  > cd $DEMO
