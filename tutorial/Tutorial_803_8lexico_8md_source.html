6 As discussed above, HiFST uses a lexicographic semiring (\ref basic_scores, [\ref Roark2011])
7 of two tropical weights. In each arc of a lattice generated by HiFST,
8 the first weight (G+M) contains the correct score (translation grammar score + language model score). The second weight G only contains the
9 translation grammar score. An example is repeated here:

23 The advantage of using the lexicographic semiring to represent (G+M,G)
24 weights is that the language model score can be substracted very
25 efficiently. HiFST binaries do this mapping internally with an
26 [fstmap](http://www.openfst.org/twiki/bin/view/FST/ArcMapDoc)
27 operation. The result is a WFSA whose weights contain only the translation
28 grammar scores (G,G). The lexmap tool can be used to do this mapping, as follows, yielding lexicographic weights (G,G):
29 
30  > zcat output/exp.baseline.outputnoprune/LATS/1.fst.gz | lexmap.O2 | fstprint | head -n 10
31  0 1 1 1 -2.609375,-2.609375
32  1 36 999999999 999999999 29.5185547,29.5185547
33  1 35 12198 12198 7.27929688,7.27929688
34  1 34 12198 12198 5.58789062,5.58789062
35  1 33 9227 9227 2.56347656,2.56347656

42 Using this facility to remove language model scores,
43 the HiFST applylm tool can be used to rescore lattices under a different language model than was used in first-pass translation. Operations are as follows:
44 
45  -# A lattice with lexicographic (G+M,G) weights is loaded
46  -# Weights are converted to (G,G) via fstmap
47  -# The new language model(s) are applied via composition under the lexicographic semiring, with optional scale factors and word insertion penalties. The new WFSAs weights are of the form (G+M',G), where M' are the new language model weights.
48  -# The reweighted WFSA is written to disk, with either lexicographic or standard tropical weights
49 
50 The following example uses applylm to rescore lattices generated with
51 almost no pruning (`output/exp.baseline.outputnoprune/LATS`). Rescoring
52 uses the same 4-gram language model originally used to generate the
53 lattice, but with a different scale factor (`lm.scale=0.9`).
54 
55  > applylm.O2 --config=configs/CF.baseline.outputnoprune.lmrescore &> log/log.lmrescore
