1 Multi-dimensional MERT {#rsearch}
2 =================================
3 
4 This section describes how to perform a multi-dimensional reverse
5 search [\ref Fukuda2004, \ref Waite2014, \ref Waite2015] over
6 N-best lists. Quoting from the abstract of [\ref Waite2015], multi-dimensional MERT is
7 
8  > ... a polynomial-time generalisation of line optimisation that
9  > computes the error surface over a plane embedded in parameter
10  > space. The description of this algorithm relies on convex
11  > geometry, which is the mathematics of polytopes and their faces.
12 
13 Chapters 5 and 6 of [\ref Waite2014] give a complete presentation of
14 the underlying theory. This tutorial reviews the procedure used to
15 carry out the two-dimensional MERT described in Section 4.2 of [\ref
16 Waite2015] and Section 6.4 of [\ref Waite2014]. This second reference also
17 explains some of the steps that are carried out within the procedure.
18 
19 For this example we
20 use the reverse search implementation of C. Weibel
21 [\ref Weibel2010]. Because his implementation has a GPL license it must be
22 downloaded and compiled separately. The version used in the following
23 example is
24 [1.6.2f](https://sites.google.com/site/christopheweibel/research/minksum/MINKSUM_1.6.2f.tar.gz?attredirects=0).
25 **The scripts used in the example assume that the shell variable $MINK_SUM
26 points to the bin directory of this tool.**
27 
28 The following script will perform a mulit-dimensional MERT over 150 sentences.
29 *N.B.* Before running reverse search it is necessary to generate the N-Best lists.
30 To do this, follow Steps 1, 2, and 3 in the tutorial section \ref mert with `M=150`, and then run
31 
32  > scripts/reverse_search.sh
33 
34 Let us examine the key steps in this script.
35 
36 \section affine_project Step 1. Feature Projection
37 
38 Multi-dimensional MERT casts the line optimisation procedure as the
39 affine projection of a linear space to a lower dimensional affine
40 subspace. Within this lower dimensional affine subspace it is possible
41 to use the reverse search algorithm to enumerate each set of
42 hypotheses that can generated from the N-best lists.
43 For this example a 3-dimensional affine subspace is constructed where
44 the first feature dimension is the affine component, and the other two
45 features map to the second `f_1(e)` and fourth `f_3(e)` features in the
46 feature vectors `F(e)` contained in the N-best lists.
47 The projection is performed as described in [\ref Waite2014, Section
48 6.4]. The initial parameter value, \f$w^{(0)}\f$ is the 12-dimensional
49 parameter vector used to generate the N-best list (see
50 `configs/CF.mert.hyps` and \ref weight_feature_vector_examples).
51 
52 A `3x12` dimensional matrix \f$A_{3,12}\f$ is formed from \f$w^{(0)}\f$ as described in [\ref
53 Waite2014, Section 6.4] so that any 12-dimensional feature vector \f$f\f$
54 can be projected into the subspace as \f$A_{3,12} f\f$. This is done using the following script:
55 
56  > zcat output/exp.mert/nbest/VECFEA/1.vecfea.gz | ./scripts/affine_project.py $FW
57  [
58  [[-48.2233,-10.8672,16.0000],[-47.3144,-12.8613,17.0000],[-50.7771,-9.6670,16.0000], ...
59 
60 The output format is required by the reverse search implementation and
61 consists of 3-dimensional row vectors. In this example there are 100
62 vectors, one for each entry in the 100-best list.
63 The reverse search algorithm requires a well known starting parameter
64 to begin its search, and the implementation uses the parameter
65 \f$[1,1,1]\f$ as its starting point. However we wish the search procedure to start from the projection of \f$w^{(0)}\f$.
66 To enforce this, the affine projection script subtracts 1 from the second and
67 fourth parameter to force the reverse search implementation to use the
68 starting parameter `[1,0,0]`. The affine projection script also negates
69 the feature vectors because the convention of reverse search is to
70 maximise scores, as opposed to the cost minimization convention of
71 HiFST.
72 
73 \section convex_hull Step 2. Convex Hulls of Projected Features
74 
75 The following line computes the convex hull from the set of projected feature vectors. This is performed separately for each N-Best list:
76 
77  > SETSIZE=150
78  > for i in `seq 1 $SETSIZE`; do cat output/exp.mert/polytope/$i.txt | $MINK_SUM/convexHull -d > output/exp.mert/hull/$i.txt;done
79 
80 The result is a filtered set of feature vectors that form vertices of
81 the convex hull for each sentence. Since any input feature vector that is an interior point is discarded, these files are shorter than the original N-best lists.
82 These vertices are then compiled
83 into an single file for input into the reverse search tool.
84 
85 \section reverse_search_op Step 3. Reverse Search
86 
87 The reverse search is performed as follows:
88 
89  > mkdir -p output/exp.mert/reverse_search
90  > cd output/exp.mert/reverse_search
91  > $MINK_SUM/minkSumForkGrid -c -n 12 < ../minksumin.txt
92 
93 The `-c` option instructs the reverse search tool to compute the full
94 normal cone associated with each result. This option can be omitted
95 for faster computation.
96 
97 The `-n 12` option is the number of child processes to be used for
98 parallel computation. In this case 12 are spawned.
99 
100 The output of the tool is contained in the directory
101 `output/exp.mert/reverse_search`. In this directory is an output file
102 for each child process. Each line is these output files corresponds to
103 a normal cone. Let us examine one line from an output file
104 
105  [13,23,19,16,6,19,14,16,9,22,15,16,10,16,14,4,11,10,16,9,1,11,12,12,16,15,8,1,6,12,0,20,13,7,5,8,3,26,11,17,9,9,7,19,15,7,6,10,19,2,17,8,7,8,10,14,6,9,18,11,6,9,12,5,10,22,21,17,4,1,13,7,21,13,17,7,11,6,11,12,15,15,19,14,15,30,20,9,9,5,15,14,10,16,5,21,12,8,4,6,18,16,8,9,10,15,10,8,8,12,2,13,20,3,9,23,12,17,8,10,12,17,19,9,20,7,16,12,10,13,9,9,11,18,23,21,18,14,12,12,15,11,19,22,15,20,11,13,1,2] : [-12442.6,-1992.01,3084] : [38.187,303.022,-2189.79] : [[-0,-14.3453,10.706,1],[-0,8.69,-8.3183,-1],[-0,1.568,-0.1943,-0],[-0,-1.2371,0.1592,-0]]
106 
107 This line is divided into four fields separated by the colon character `:`
108 
109 - The first field represents the set of hypotheses maximised by the parameters in this cone. It is a vector of 150 elements, with each element corresponding to an index to a vertex in the convex hull associated with each input sentence
110 - The second field is the sum of all the vertices
111 - The third field is a parameter interior to the normal cone which maximises all the vertices in the first field
112 - The fourth field is an optional field associated with the `-c` option. It is the set of rays that define the cone
113 
114 The parameter vector (the third field) can be used to rescore the
115 N-Best lists by their projected feature vectors: each
116 parameter vector will return the hypotheses identified by the first
117 field (barring numerical errors).
118 
119 Due to the geometry of a normal cone, the parameter vectors can be mapped from 3 to 12 dimensions, as follows:
120 - Parameter vectors for which the first element of the parameter vector is negative are rejected
121 - The parameter vector is scaled by the first element (\f$f_i \leftarrow f_i / f_1\f$)
122 - A new 12-dimensional parameter vector is created by adding the second and third element of the scaled projected feature vector to the second and fourth elements of the initial parameter
123 
124 \section reverse_search_reranking Step 4. Reranking
125 
126 Once the reverse search is complete, we should complete a sanity test
127 to ensure that the parameters found by the reverse search correctly
128 rerank the N-best lists.
129 
130 The first step is to map the vertices in the
131 convex hull to the feature vectors in the original N-best list.
132 
133  > scripts/map_vertices.py output/exp.mert/polytope output/exp.mert/hull > output/exp.mert/mapping.txt
134 
135 The result of this script is a pickled python map object that provides
136 the mapping between index of a vertex in a convex hull to the index of
137 the corresponding feature vector in the N-best list. Using this
138 mapping between convex hull and the indices of the N-best list, the sanity test is
139 executed by the following script:
140 
141  > cat output/exp.mert/reverse_search/result.* |\
142  scripts/reverse_search_test.py output/exp.mert/mapping.txt output/exp.mert/nbest/VECFEA $GW > log/log.reverse.search.sanity
143 
144 The 3-dimensional parameter vectors found by the reverse search are mapped up to 12 dimensions (as described above) and used
145 to re-rank the 150 N-best lists. The index of the top scoring hypotheses of the reranked
146 lists should match the mapped index from the first field of the
147 reverse search output. If the indices do not match, the sanity test
148 script prints a line of the form:
149 
150  Discrepancy! For input sentence 31 featue vector 2 is the top scorer
151 
152 Due to floating point errors there will be some N-best lists that are
153 reranked incorrectly.
154 For reference, using 150 input sentences with 100-best lists generated
155 346,526 parameters. Of these parameters 185,475 were accepted for
156 testing (i.e. had \f$f_1 > 0\f$), which means that 27,821,250 (=150 * 185,475) reranking tests were
157 performed. Out of these tests 28,282 rerankings had the wrong top
158 scoring hypothesis, which is approximately 1% of all the reranking
159 tests performed.
