1 Neural Machine Translation {#nmt}
2 =================================
3 
4 This section describes how to prepare lattices and RTNs produced by HiFST for
5 our neural machine translation (NMT) tool
6 [SGNMT](http://ucam-smt.github.io/sgnmt/html/)
7 [\ref Stahlberg2016] which is an extension of the NMT implementation in the
8 [Blocks](http://blocks.readthedocs.io/en/latest/)
9 framework. SGNMT supports lattice, RTN, and n-best rescoring with single
10 or ensembled NMT, integration of language models and much more. Examples can
11 be found on the [SGNMT tutorial page](http://ucam-smt.github.io/sgnmt/html/tutorial.html).

14 \section prepare_lattices Preparing Lattices for SGNMT
15 
16 Translation lattices produced by HiFST use the **lexicographic semiring of two tropical weights** [\ref Roark2011] \(see \ref basic_scores\).
17 However, SGNMT requires lattices with standard arcs. HiFST provides the `lexmap` tool to convert lexicographic to standard arcs. Additionally,
18 we usually apply epsilon removal, determinisation, minimisation, and weight pushing to the translation lattices before passing them
19 through to SGNMT [\ref Stahlberg2016]. We also map HiFST special symbols to epsilon in order to remove them from the processed lattices.
20 The full pipeline looks like that:
21 
22  > cat hifst_special_symbol_to_eps_map
23  999999991 0
24  999999992 0
25  999999993 0
26  999999994 0
27  999999995 0
28  999999996 0
29  999999997 0
30  999999998 0
31  999999999 0
32  100000000 0
33  > zcat output/exp.baseline/LATS/1.fst.gz | lexmap.${TGTBINMK}.bin --action=lex2std | \
34  fstrelabel -relabel_ipairs=hifst_special_symbol_to_eps_map -relabel_opairs=hifst_special_symbol_to_eps_map | \
35  fstrmepsilon | \
36  fstdeterminize | \
37  fstminimize | \
38  fstmap -map_type=to_log | \
39  fstpush --push_weights > output/exp.baseline/LATS.sgnmt/1.fst
40 
41 The `scripts/` directory in the [SGNMT tutorial data](http://ucam-smt.github.io/sgnmt/html/tutorial.html) contains the script
42 `create_fst_directory.sh` which prepares a complete HiFST lattice directory for SGNMT.
43 
44 Note that for large lattices, determinisation is not possible because \ref OpenFst [Determinize](http://www.openfst.org/twiki/bin/view/FST/DeterminizeDoc)
45 requires too much memory. In this case, you can skip epsilon removal, determinisation, and minimisation, and use SGNMT's
46 [nfst predictor](http://ucam-smt.github.io/sgnmt/html/predictors.html).

48 \section prepare_rtns Preparing RTNs for SGNMT
49 
50 SGNMT supports decoding on the search space spanned by an RTN. Arc expansion (i.e. replacing a non-terminal arc with a FST) is done
51 on-the-fly only if the search algorithm arrives at this arc (late expansion). Like lattices, RTNs need to be converted from the
52 lexicographic semiring using the `lexmap` tool, but without changing the directory structure.
53 The `scripts/` directory in the [SGNMT tutorial data](http://ucam-smt.github.io/sgnmt/html/tutorial.html) contains the script
54 `create_rtn_directory.sh` to do that.
