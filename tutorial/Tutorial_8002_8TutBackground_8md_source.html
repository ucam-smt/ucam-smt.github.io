31 \section Setup_configs Configuration Files and Command Line Options
32 
33 HiFST uses the Boost libraries which provide support for command line options and
34 [configuration files](http://www.boost.org/doc/libs/1_55_0/doc/html/program_options/overview.html).
35 For example, the
36 following options could be provided on the command line:
37 
38  --hifst.prune=9 --hifst.replacefstbyarc.nonterminals=X,V
39 
40 Alternatively, they could be specified in a configuration file either as
41 
42  hifst.prune=9
43  hifst.replacefstbyarc.nonterminals=X,V
44 
45 or as
46 
47  [hifst]
48  prune=9
49  replacefstbyarc.nonterminals=X,V
50 
51 As you work through the tutorial, please read the comments in the
52 config files which explain some of the processing options. For example, see `configs/CF.baseline`:
53 
54 
55  # Basic HiFST configuration for Russian-English translation
56  # Translation uses a 4gram language model with a shallow-1 grammar
57  # as described in
58  # J. Pino, et al. The University of Cambridge Russian-English System at
59  # WMT13. http://aclweb.org/anthology//W/W13/W13-2225.pdf
60 
61  range=1:2
62  # range of line numbers in the source text file to be translated
63 
64  [source]
65  load=RU/RU.tune.idx
66  # path and filename of the source text file
67 
68  [target]
69  store=output/exp.baseline/hyps
70  # path and filename of the target text file; translations are written to this file
71 
72  [hifst]
73  lattice.store=output/exp.baseline/LATS/?.fst.gz
74  # specifies the path and filename into which translation lattices are
75  # written. Note the use of the placeholder '?' in the argument
76  # .../LATS/?.fst.gz . The placeholder is replaced by the line
77  # number of sentence being translated, e.g. so that
78  # .../LATS/2.fst.gz is a WFST containing translations of the second
79  # line in the source text file. Note also the use of the '.gz'
80  # extension: when this is provided, lattices are written as gzipped
81  # files.
82 
83  prune=9
84  # translation lattices are pruned using OpenFST pruning operations
85  # prior to saving to disk. The parameter provided is the pruning
86  # threshold. The default is 3.40282347e+38 which effectively means
87  # no pruning.
88 
89  replacefstbyarc.nonterminals=X,V
90  # Specifies non-terminals in the grammar, apart from the top-level
91  # non-terminal S. HiFST will treat X and V as non-terminals;
92  # in addition, other symbols Xn and Vn (e.g. X1 and V2) will be treated as non-terminals.
93 
94  [lm]
95  load=M/interp.4g.arpa.newstest2012.tune.corenlp.ru.idx.withoptions.mmap
96  # path and filename of the English n-gram language model in kenlm format
97  # this particular language model is a modified Kneser-Ney 4-gram language model
98  # trained for the Russian-English task at WMT13 (http://statmt.org/wmt13/)
99  # with the KenLM toolkit (http://kheafield.com/code/kenlm/)
100 
101  [grammar]
102  load=G/rules.shallow.gz
103  # path and filename of the Hiero translation grammar to load and use in translation
104  # this particular grammar is a shallow grammar: when CYK parsing the source side,
105  # hierarchical rules can only be concatenated with the glue rule and not combined
106  # together with another hierarchical rules.

113 \section wmaps Word Maps
114 
115 HiFST uses [symbol tables](http://www.openfst.org/twiki/bin/view/FST/FstAdvancedUsage#Symbol_Tables)
116 as provided by \ref OpenFst to map between source and target
117 language text and the integer representation used internally by
118 the decoder. See the \ref OpenFst [Quick Tour](http://www.openfst.org/twiki/bin/view/FST/FstQuickTour) for a
119 discussion of the use of symbol tables.
120 
121 Integer mappings for English and Russian are in the directory wmaps/ :
122 
123  wmaps/wmt13.en.wmap
124  wmaps/wmt13.ru.wmap
125 
126 Note that HiFST reserves the integers `1` and `2` for the sentence-start and sentence-end symbols. `0` is the \ref OpenFst epsilon symbol. `999999998` is the OOV symbol.
127 
128 The format of the wordmap files is straightforward, e.g.
129 
130  > head wmaps/wmt13.en.wmap
131  <epsilon> 0
132  <s> 1
133  </s> 2
134  the 3
135  , 4
136  . 5
137  of 6
138  to 7
139  and 8
140  in 9

184 \section tgrammars Translation Grammars
185 HiFST uses Synchronous Context-Free Grammars (SCFGs) for translation.
186 A full Hiero and a Shallow-1 translation grammar are provided in the `G/` directory:
187 
188  G/rules.shallow.gz : Shallow-1 hiero grammar with scalar scores
189 
190 We also provide a grammar with raw, unweighted feature vectors:
191 
192  G/rules.shallow.vecfea.gz : Shallow-1 hiero grammar with feature vectors
193 
194 There is also a grammar provided for the true-casing example (\ref true_casing)
195 
196  G/tc.unimap
197 
198 \subsection rules Grammar File Formats
199 
200 In the grammar file, each line represents a rule. The rule format is:
201 
202  LHS RHS_SOURCE RHS_TARGET FEA_1 [FEA_2 FEA_3 FEA_4 ...]
203 
204 where
205 
206  LHS = the left hand side of the rule
207  RHS_SOURCE = the source-language part of the right hand side of the rule
208  RHS_TARGET = the target-language part of the right hand side of the rule
209  FEA_i = the i-th component of the feature vector associated with the rule
210 
211 The left hand side of a rule is a non-terminal symbol (in uppercase).
212 The right hand side is a pair of terminal and non-terminal
213 symbol sequences in the source and target languages.
214 
215 \subsection tgrammars_formats_fea Feature Vectors
216 
217 Scores are assigned to rules as the dot product of a rule-specific feature vector
218 and a weight vector ([\ref Chiang2007] and see the discussion in \ref lmert). This
219 computation can be done offline, in which case the feature for every rule in the grammar is a
220 1-dimensional scalar. Alternatively, the decoder can be provided with a weight vector which is applied
221 to the feature vectors while loading the grammar.
222 For example, the grammar `G/rules.shallow.gz` provided in this tutorial the following set of weights was found via LMERT tuning (\ref lmert ):
223 
224  0.697263,0.396540,2.270819,-0.145200,0.038503,29.518480,-3.411896,-3.732196,0.217455,0.041551,0.060136
225 
226 These can be applied to the Shallow-1 grammar, as follows:
227 
228  > gzip -d -c G/rules.shallow.vecfea.gz | head -3
229  V 3 4 0.223527 0.116794 -1 -1 0 0 0 0 -1 1.268789 0.687159
230  V 3 4_3 3.333756 0.338107 -2 -1 0 0 0 0 -1 1.662178 3.363062
231  V 3 8 3.74095 3.279819 -1 -1 0 0 0 0 -1 3.741382 2.271445
232 
233  > GW=0.697263,0.396540,2.270819,-0.145200,0.038503,29.518480,-3.411896,-3.732196,0.217455,0.041551,0.060136
234 
235  > zcat G/rules.shallow.vecfea.gz | scripts/weightgrammar -w=$GW | head -3
236  V 3 4 -2.046860955276
237  V 3 4_3 -1.884009085882
238  V 3 8 1.857985226112
239 
240 and this should agree with the scalar-valued version of the grammar:
241 
242  > zcat G/rules.shallow.gz | head -3
243  V 3 4 -2.046860955276
244  V 3 4_3 -1.884009085882
245  V 3 8 1.857985226112
246 
247 **Note** that the unweighted feature vector contains all features
248  needed to compute the score of a translation, *except* for the
249  contributions from the language model(s).

252 \subsection tgrammars_formats_nt Non-Terminals
253 
254 In translation, a non-terminal `X` on the right hand side can be
255 rewritten by any rule whose left hand side is `X`. HiFST places
256 no restrictions on the definition of terminal and non-terminal
257 sequences in an SCFG rule; similarly, there are no constraints on how
258 many non-terminal symbols can be used in a rule (i.e. there are no constraints on order of the
259 SCFG). However using a full Hiero grammar can lead to slow
260 translation, and this tutorial discusses several strategies for
261 pruning in translation and for translation grammar pruning.
262 
263 Here is a small sample translation grammar
264 
265  M 434_M 1462_8_M -1.81842
266  M 7_M 9_3_M -0.735445
267  M V V -0
268  S S_X S_X 0.05768
269  S X X -0
270  V 10806 1411 1.16623
271  V 164_M_60 78_M_8 -0.226464
272  V 164_M2_60_M1 78_M1_8_M2 -0.226464
273  V 21_591 39_258_8 -0.510102
274  V 24 3_54 -2.50252
275  V 274_M_4 709_9_3_M -0.589246
276  V 5 6 -1.81729
277  V 7_1689 9_741_8 0.438945
278  V 8 23 -1.46604
279  X 1 1 -2.5598
280  X 2 2 -2.5598
281  X V V -0
282  D 1775 <dr> 10.4327
283  S S_D_X S_D_X 0.11536
284 
285 
286 This grammar has four non-terminal symbols: `S`, `M`, `V` and `D`, and a 1-dimensional weight. The terminal symbols are integers, corresponding to words in the source and target language word maps (\ref wmaps). The symbol `<dr>`
287 is a special symbol used by HiFST to represent an empty word, to indicate deletion.
288 
289 
290 As an example, for rule
291 
292  V 164_M_60 78_M_8 -0.226464
293 
294 we have
295 
296  LHS = V
297  RHS_SOURCE = 164_M_60
298  RHS_TARGET = 78_M_8
299  WEIGHT_1 = -0.226464
300 
301 With this rule the decoder can rewrite the non-terminal `V` by replacing it by `164_M_60`
302 in the source language and by `78_M_8`
303 in the target language; the rule is applied with
304 a score of -0.226464. Similarly, rule `V 5 6 -1.81729`
305 replaces `V` by the word "5" in the source-language and with the word
306 "6" in the target-language, with a score of -1.81729.
307 
308 Indices on non-terminals indicate alignment within rules with more than one non-terminal. For example, for rule
309 
310  V 164_M2_6_M1 78_M1_8_M2 -0.226464
311 
312 the `M` non-terminals on both language sides are indexed by 1 or 2; that is, `M1` in the source
313 language is linked with `M1` in the target language, and `M2` in the
314 source language is linked with `M2` in the target.
315 Strictly speaking, `M2` is not a non-terminal in the above example: it is the second
316 instance of the non-terminal `M` in the rule.
317 Obviously this is a reordering rule.
318 
319 In general we use different types of rule to distinguish different
320 translation cases and obtain a finer-grained model. In this example, the `S`
321 rules are doing something very similar to the glue rules used in
322 Hiero-style systems; the `M` rules can be regarded as monotonic
323 translation rules; the `V` rules can be regarded as reordering
324 translation rules and phrasal translation rules; and the `D` rule can be
325 regarded as an explicit operation of word deletion.
326 
327 Note that there is a straight-forward correspondence between the
328 Hiero-style rule notation introduced by [\ref Chiang2007]
329 and the HiFST rule file format (for rules with
330 one-dimensional weights). For example, the HiFST grammar file entries
331 
332  V 164_M2_6_M1 78_M1_8_M2 -0.226464
333  S S_X S_X 0.05768
334 
335 can be written as
336 
337  V -> < 164 M2 6 M1 , 78 M1 8 M2 > / -0.226464
338  S -> < S X , S X > / 0.05768

341 \subsection tgrammars_shallow Shallow-N Translation Grammars
342 
343 Shallow grammars [\ref deGispert2010] can be used to control the degree of nesting allowed within an hierarchical grammar.
344 For example, for a Shallow-1 Grammar, variables in a rule can be substituted only by phrases. That is,
345 hierarchical rules can be used only once to generate words; once a hierarchical rule is used, translation relies on glue rules to cover longer source spans.
346 
347 Formally, we use W to denote the set of terminals, and S
348 and X to denote two non-terminals. A simple
349 Hiero-style grammar can be defined to be:
350 
351  S -> X, X
352  S -> S X, S X
353  X -> a, b where a, b \in ({X} union W)^+
354 
355 We can transform this into a Shallow-1 grammar as
356 
357  S -> X, X
358  S -> S X, S X
359  Y -> a, b where a, b \in {W}^+
360  X -> u, v where u, v \in {{Y} union W}^+
361 
362 Here `Y` is introduced to handle phrasal translations. The variables in rule `X -> u, v` can only be substituted with the `Y` rules.
363 
364 In some cases it is desirable to allow more complex movement in
365 translation, such as complex structure movements in Chinese-English
366 translation. For this we can use a generalisation of the simple
367 Shallow-1 grammar, called Shallow-N grammars. These grammars allow hierarchical
368 rules to be applied up to N times.
369 For example, below is the form of a Shallow-2 grammar.
370 
371  S -> X, X
372  S -> S X, S X
373  Y^0 -> a^0, b^0 where a^0, b^0 \in {W}^+
374  Y^1 -> a^1, b^1 where a^1, b^1 \in {{Y^0} union W}^+
375  X -> u, v where u, v \in {{Y^1} union W}^+
376 
377 
378 The Shallow-2 grammar introduces `Y^0` and `Y^1` to handle hierarchical
379 rule application at two levels within a derivation. For more detailed
380 description of Shallow-n grammars, please refer to the HiFST paper [\ref deGispert2010].
