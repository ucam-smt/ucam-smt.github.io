30 \section Setup_configs Configuration Files and Command Line Options

36 
37  --hifst.prune=9 --hifst.replacefstbyarc.nonterminals=X,V

41  hifst.prune=9

77 \section wmaps Word Maps
78 
79 HiFST uses [symbol tables](http://www.openfst.org/twiki/bin/view/FST/FstAdvancedUsage#Symbol_Tables)
80 as provided by \ref OpenFst to map between source and target
81 language text and the integer representation used internally by
82 the decoder. See the \ref OpenFst [Quick Tour](http://www.openfst.org/twiki/bin/view/FST/FstQuickTour) for a
83 discussion of the use of symbol tables.
84 
85 Integer mappings for English and Russian are in the directory wmaps/ :
86 
87  wmaps/wmt13.en.wmap
88  wmaps/wmt13.en.all.wmap (a much larger version of wmaps/wmt13.en.wmap)
89  wmaps/wmt13.ru.wmap
90  wmaps/wmt13.ru.all.wmap (a much larger version of wmaps/wmt13.ru.wmap)
91 
92 Note that HiFST reserves the integers 1 and 2 for the sentence-start and sentence-end symbols. 0 is the \ref OpenFst epsilon symbol.
93 
94 The format of the wordmap files is straightforward, e.g.
95 
96  > head wmaps/wmt13.en.wmap
97  <epsilon> 0
98  <s> 1
99  </s> 2
100  the 3
101  , 4
102  . 5
103  of 6
104  to 7
105  and 8
106  in 9

204 \subsection rules Grammar File Formats
205 
206 In the grammar file, each line represents a rule. The rule format is:
207 
208  LHS RHS_SOURCE RHS_TARGET FEA_1 [FEA_2 FEA_3 FEA_4 ...]
209 
210 where
211 
212  LHS = the left hand side of the rule
213  RHS_SOURCE = the source-language part of the right hand side of the rule
214  RHS_TARGET = the target-language part of the right hand side of the rule
215  FEA_i = the i-th component of the feature vector associated with the rule
216 
217 The left hand side of a rule is a non-terminal symbol (in uppercase).
218 The right hand side is a pair of terminal and non-terminal
219 symbol sequences in the source and target languages.
220 
221 \subsection tgrammars_formats_fea Feature Vectors
222 
223 Scores are assigned to rules as the dot product of a rule-specific feature vector
224 and a weight vector ([\ref Chiang2007] and see the discussion in \ref mert). This
225 computation can be done offline, in which case the feature for every rule in the grammar is a
226 1-dimensional scalar. Alternatively, the decoder can be provided with a weight vector which is applied
227 to the feature vectors while loading the grammar.
228 
229 For example, the grammar `G/rules.shallow.gz` provided in this tutorial the following set of weights was found via LMERT tuning (see \ref mert ):
230 
231  0.697263,0.396540,2.270819,-0.145200,0.038503,29.518480,-3.411896,-3.732196,0.217455,0.041551,0.060136
232 
233 These can be applied to the Shallow-1 grammar, as follows:
234 
235  > gzip -d -c G/rules.shallow.vecfea.gz | head -3
236  V 3 4 0.223527 0.116794 -1 -1 0 0 0 0 -1 1.268789 0.687159
237  V 3 4_3 3.333756 0.338107 -2 -1 0 0 0 0 -1 1.662178 3.363062
238  V 3 8 3.74095 3.279819 -1 -1 0 0 0 0 -1 3.741382 2.271445
239 
240  > GW=0.697263,0.396540,2.270819,-0.145200,0.038503,29.518480,-3.411896,-3.732196,0.217455,0.041551,0.060136
241 
242  > zcat G/rules.shallow.vecfea.gz | scripts/weightgrammar -w=$GW | head -3
243  V 3 4 -2.046860955276
244  V 3 4_3 -1.884009085882
245  V 3 8 1.857985226112
246 
247 and this should agree with the scalar-valued version of the grammar:
248 
249  > zcat G/rules.shallow.gz | head -3
250  V 3 4 -2.046860955276
251  V 3 4_3 -1.884009085882
252  V 3 8 1.857985226112
253 
254 **Note** that the unweighted feature vector contains all features
255  needed to compute the score of a translation, *except* for the
256  contributions from the language model(s).

259 \subsection tgrammars_formats_nt Non-Terminals
260 
261 In translation, a non-terminal `X` on the right hand side can be
262 rewritten by any rule whose left hand side is `X`. HiFST places
263 no restrictions on the definition of terminal and non-terminal
264 sequences in an SCFG rule; similarly, there are no constraints on how
265 many non-terminal symbols can be used in a rule (i.e. there are no constraints on order of the
266 SCFG). However using a full Hiero grammar can lead to slow
267 translation, and this tutorial discusses several strategies for
268 pruning in translation and for translation grammar pruning.
269 
270 Here is a small sample translation grammar
271 
272  M 434_M 1462_8_M -1.81842
273  M 7_M 9_3_M -0.735445
274  M V V -0
275  S S_X S_X 0.05768
276  S X X -0
277  V 10806 1411 1.16623
278  V 164_M_60 78_M_8 -0.226464
279  V 164_M2_60_M1 78_M1_8_M2 -0.226464
280  V 21_591 39_258_8 -0.510102
281  V 24 3_54 -2.50252
282  V 274_M_4 709_9_3_M -0.589246
283  V 5 6 -1.81729
284  V 7_1689 9_741_8 0.438945
285  V 8 23 -1.46604
286  X 1 1 -2.5598
287  X 2 2 -2.5598
288  X V V -0
289  D 1775 <dr> 10.4327
290  S S_D_X S_D_X 0.11536
291 
292 
293 This grammar has four non-terminal symbols: `S`, `M`, `V` and `D`, and a 1-dimensional weight. The terminal symbols are integers, corresponding to words in the source and target language word maps (\ref wmaps). The symbol `<dr>`
294 is a special symbol used by HiFST to represent an empty word, to indicate deletion.
295 
296 
297 As an example, for rule
298 
299  V 164_M_60 78_M_8 -0.226464
300 
301 we have
302 
303  LHS = V
304  RHS_SOURCE = 164_M_60
305  RHS_TARGET = 78_M_8
306  WEIGHT_1 = -0.226464
307 
308 With this rule the decoder can rewrite the non-terminal `V` by replacing it by `164_M_60`
309 in the source language and by `78_M_8`
310 in the target language; the rule is applied with
311 a score of -0.226464. Similarly, rule `V 5 6 -1.81729`
312 replaces `V` by the word "5" in the source-language and with the word
313 "6" in the target-language, with a score of -1.81729.
314 
315 Indices on non-terminals indicate alignment within rules with more than one non-terminal. For example, for rule
316 
317  V 164_M2_6_M1 78_M1_8_M2 -0.226464
318 
319 the `M` non-terminals on both language sides are indexed by 1 or 2; that is, `M1` in the source
320 language is linked with `M1` in the target language, and `M2` in the
321 source language is linked with `M2` in the target.
322 Strictly speaking, `M2` is not a non-terminal in the above example: it is the second
323 instance of the non-terminal `M` in the rule.
324 Obviously this is a reordering rule.
325 
326 In general we use different types of rule to distinguish different
327 translation cases and obtain a finer-grained model. In this example, the `S`
328 rules are doing something very similar to the glue rules used in
329 Hiero-style systems; the `M` rules can be regarded as monotonic
330 translation rules; the `V` rules can be regarded as reordering
331 translation rules and phrasal translation rules; and the `D` rule can be
332 regarded as an explicit operation of word deletion.
333 
334 Note that there is a straight-forward correspondence between the
335 Hiero-style rule notation introduced by [\ref Chiang2007]
336 and the HiFST rule file format (for rules with
337 one-dimensional weights). For example, the HiFST grammar file entries
338 
339  V 164_M2_6_M1 78_M1_8_M2 -0.226464
340  S S_X S_X 0.05768
341 
342 can be written as
343 
344  V -> < 164 M2 6 M1 , 78 M1 8 M2 > / -0.226464
345  S -> < S X , S X > / 0.05768

348 \subsection tgrammars_shallow Shallow-N Translation Grammars
349 
350 Shallow grammars [\ref deGispert2010] can be used to control the degree of nesting allowed within an hierarchical grammar.
351 For example, for a Shallow-1 Grammar, variables in a rule can be substituted only by phrases. That is,
352 hierarchical rules can be used only once to generate words; once a hierarchical rule is used, translation relies on glue rules to cover longer source spans.
353 
354 Formally, we use W to denote the set of terminals, and S
355 and X to denote two non-terminals. A simple
356 Hiero-style grammar can be defined to be:
357 
358  S -> X, X
359  S -> S X, S X
360  X -> a, b where a, b \in ({X} union W)^+
361 
362 We can transform this into a Shallow-1 grammar as
363 
364  S -> X, X
365  S -> S X, S X
366  Y -> a, b where a, b \in {W}^+
367  X -> u, v where u, v \in {{Y} union W}^+
368 
369 Here `Y` is introduced to handle phrasal translations. The variables in rule `X -> u, v` can only be substituted with the `Y` rules.
370 
371 In some cases it is desirable to allow more complex movement in
372 translation, such as complex structure movements in Chinese-English
373 translation. For this we can use a generalisation of the simple
374 Shallow-1 grammar, called Shallow-N grammars. These grammars allow hierarchical
375 rules to be applied up to N times.
376 For example, below is the form of a Shallow-2 grammar.
377 
378  S -> X, X
379  S -> S X, S X
380  Y^0 -> a^0, b^0 where a^0, b^0 \in {W}^+
381  Y^1 -> a^1, b^1 where a^1, b^1 \in {{Y^0} union W}^+
382  X -> u, v where u, v \in {{Y^1} union W}^+
383 
384 
385 The Shallow-2 grammar introduces `Y^0` and `Y^1` to handle hierarchical
386 rule application at two levels within a derivation. For more detailed
387 description of Shallow-n grammars, please refer to the HiFST paper [\ref deGispert2010].
