9 \subsection lmert_run HiFST_lmert
10 
11 This HiFST release includes an implementation of LMERT [\ref Waite2012]. The
12 script `HiFST_lmert` runs several iterations of lattice generation and
13 parameter estimation using the `lmert` tool.
14 
15 Prior to running the script, make sure to download and uncompress the
16 LM `interp.4g.arpa.newstest2012.tune.corenlp.ru.idx.withoptions.mmap`
17 into the `M/` directory (see \ref build).
18 
19  > scripts/HiFST_lmert
20 
21 This script runs 4 iterations of LMERT, with each iteration consisting of the four steps that follow in the sections below.
22 In our experience, on a X86_64 Linux computer with 4 2.8GHz CPUs and 24GB RAM, each iteration takes ca. 3 hours.
23

59 Note also that the script `HiFST_lmert` can be modified to perform
60 LMERT over only the first (e.g.) 100 tuning set sentences. This can
61 be done for debugging / demonstration, in that processing will be much
62 faster, although the estimated parameters will not be as robust.

66 \subsection lmert_hyps 1. Hypotheses for LMERT
67 
68 Note that this step is also done in MERT (\ref mert), although the settings here are slightly different.
69 
70 The following command is run at iteration `$it` , and will generate lattices for `$M` files.
71 
72 - Input:
73  - `$it` -- lmert iteration (1, 2, ...)
74  - `$M` -- number of sentences to process
75  - `RU/RU.tune.idx` -- tuning set source language sentences
76  - `G/rules.shallow.vecfea.all.gz` -- translation grammar
77  - `M/interp.4g.arpa.newstest2012.tune.corenlp.ru.idx.union.mmap` -- target language model
78  - language model and translation grammar feature weights, provided via command line option
79 - Output
80  - `output/exp.lmert/$it/LATS/?.fst.gz` -- word lattices (WFSAs), determinized and minimized
81  - `output/exp.lmert/$it/hyps` -- translation hyps (discarded)
82 
83 The feature weights are gathered into a single vector, and set via the `--featureweights` command line option
84 
85 - The first parameter is the grammar scale factor (in the MERT demo, this is set via `lm.featureweights`)
86 - The remaining parameters are weights for the grammar features (in the MERT demo, these are set via `grammar.featureweights`)
87 
88 The initial parameters are
89 
90  FW=1.0,0.697263,0.396540,2.270819,-0.145200,0.038503,29.518480,-3.411896,-3.732196,0.217455,0.041551,0.060136
91 
92 HiFST is run in translation mode as
93 
94  > hifst.O2 --config=configs/CF.lmert.hyps --range=1:$M --featureweights=$FW --target.store=output/exp.lmert/$it/hyps --hifst.lattice.store=output/exp.lmert/$it/LATS/?.fst.gz

97 \subsection lmert_veclats 2. Guided Translation / Forced Alignment
98 
99 - Input:
100  - `$it` -- lmert iteration (1, 2, ...)
101  - `$M` -- number of sentences to process
102  - `output/exp.lmert/$it/LATS/?.fst.gz` -- word lattices (WFSAs), determinized and minimized (from \ref lmert_hyps)
103 - Output:
104  - `output/exp.lmert/$it/ALILATS/?.fst.gz` -- transducers mapping derivations to translations (e.g. Fig. 7, [\ref deGispert2010])
105  - `output/exp.lmert/$it/hyps` -- translation hyps (discarded)
106 
107 HiFST is run in alignment mode. Lattices (from `output/exp.lmert/$it/LATS/?.fst.gz`)
108 read and transformed into substring acceptors used to constrain the
109 space of alignments
110 
111  > hifst.O2 --config=configs/CF.lmert.alilats --range=1:$M --referencefilter.load=output/exp.lmert/$it/LATS/?.fst.gz --target.store=output/exp.lmert/$it/hyps --hifst.lattice.store=output/exp.lmert/$it/ALILATS/?.fst.gz
112 
113 Note the following parameters in the configuration file:
114 
115  [referencefilter]
116  prunereferenceweight=4
117  # pruning threshold to be applied to input lattice prior to alignment
118  prunereferenceshortestpath=10000
119  # extract n-best list from input lattice to use as hypotheses
120 
121 Reference lattices are read from `output/exp.lmert/$it/LATS/?.fst.gz`. For each lattice:
122 - an N-Best list of depth 10000 is extracted using `fstshortestpath`
123 - the lattices are pruned with threshold `prunereferenceweight=4`
124 - the pruned lattice is unioned with the N-Best list
125 - the resulting WFSA is transformed (after removing weights, minimization and determinization) into a substring acceptor to be used in alignment
126 
127 A simpler approach could be simply to prune the reference lattices.
128 However in practice it can be difficult to find a global pruning threshold that always
129 yields a reference lattice that is big enough, but not too big.
130 Including the n-best list ensures that there will always be a rich set of candidate hypotheses.

132 \subsection lmert_alilats 3. WFSAs with Unweighted Feature Vectors
133 
134 - Input:
135  - `$it` -- lmert iteration (1, 2, ...)
136  - `$M` -- number of sentences to process
137  - language model and translation grammar feature weights, provided via command line options
138  - `output/exp.lmert/$it/ALILATS/?.fst.gz` -- transducers mapping derivations to translations
139 - Output:
140  - output/exp.lmert/$it/VECFEA/?.fst.gz -- translation lattices (WFSAs) with unweighted feature vectors
141 
142 `alilats2splats` transforms ALILATS alignment lattices to sparse
143 vector weight lattices; see Section 2.3.1, [\ref deGispert2010] for a
144 detailed explanation. A single output WFSA with sparse vector weights
145 is written for each translation. Note that this is different from the
146 MERT case, where two N-best lists of hypotheses
147 and features are written for each translation.
148 
149 The HiFST `alilats2splats` command is
150 
151  > alilats2splats.O2 --config=configs/CF.lmert.vecfea --range=1:$M --featureweights=$FW --sparseweightvectorlattice.loadalilats=output/exp.lmert/$it/ALILATS/?.fst.gz --sparseweightvectorlattice.store=output/exp.lmert/$it/VECFEA/?.fst.gz

153 \subsection lmert_lmert 4. LMERT
154 
155 - Input:
156  - `$it` -- lmert iteration (1, 2, ...)
157  - `$M` -- number of sentences to process
158  - `output/exp.lmert/$it/VECFEA/?.fst.gz` -- translation lattices (WFSAs) with unweighted feature vectors (from \ref lmert_alilats)
159  - `EN/EN.tune.idx` -- target language references in integer format
160 - Output:
161  - `output/exp.lmert/params.$it` -- reestimated feature vector under LMERT with BLEU
162 
163 `latmert` runs as follows
164 
165  > latmert.O2 --search=random --random_axes --random_directions=28 --direction=axes --threads=24 --cache_lattices --error_function=bleu --algorithm=lmert --idxlimits=1:$M --print_precision=6 --lats=output/exp.lmert/$it/VECFEA/%idx%.fst.gz --lambda=$FW --write_parameters=output/exp.lmert/params.$it EN/EN.tune.idx

167 \subsection lmert_veclats_tst Notes on Tropical Sparse Tuple Vector Weights
168 
169 The output lattices in `output/exp.lmert/$it/lats/VECFEA` are `tropicalsparsetuple` vector weight lattices.
170 
171  > zcat output/exp.mert/1/lats/VECFEA/1.fst.gz | fstinfo | head -n 2
172  fst type vector
173  arc type tropicalsparsetuple
174 
175 The scores in these lattices are unweighted by the feature vector
176 weights, i.e. they are the raw feature scores against which L/MERT finds
177 the optimal parameter vector values. Distances under these unweighted
178 vectors do not agree with the initial translation hypotheses, e.g. the
179 shortest-path does not agree with the best translation:
